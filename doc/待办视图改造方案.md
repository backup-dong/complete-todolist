# 待办视图改造方案

> 状态：待评审  
> 更新日期：2026-07-11

## 1. 背景与问题

当前侧边栏的「待办视图」与顶部工具栏的筛选器高度重复：

| 待办视图 | 等价于 Toolbar 筛选 |
|---|---|
| 今天 | 时间范围 → 今天到期 |
| 已安排 | 时间范围 → 本周到期 |
| 已标记 | 优先级 → 高 |
| 全部 | 清空所有筛选 |

更深层的问题是：这些入口并不是真正的「待办视图」，而只是对**当前已激活清单**做过滤。用户期望待办视图是一个跨清单的全局视图。

## 2. 目标

- 把侧边栏待办视图改造为**跨所有清单聚合**的专用视图。
- 解决与 Toolbar 筛选的重复问题：待办视图下不再显示 Toolbar 的筛选和新建任务栏。
- 保留最高频操作：完成待办、查看/编辑详情、切换子任务、删除。
- 不开放拖拽排序和新建任务，避免跨清单上下文歧义。

## 3. 范围

### 3.1 本期做

- 新增专用 `TodoView` 组件，按「清单 → 分组 → 任务」三层结构展示。
- 侧边栏待办视图入口：今天、本周、全部、高优先级。
- 点击待办视图后，中间区域切换为 `TodoView`。
- 待办视图支持：
  - 完成任务（勾选）
  - 查看/编辑任务详情（弹窗）
  - 切换子任务状态
  - 删除任务
- 侧边栏显示每个待办视图的任务数量 badge。
- 点击任务详情里的清单/分组信息，可跳转回对应清单。

### 3.2 本期不做

- 待办视图下新建任务。
- 待办视图下拖拽排序。
- 待办视图下的 Toolbar 筛选与搜索（保留标题和数量即可）。
- URL 路由同步（如 `/today`），先以内置 store 状态管理。
- 批量模式（后续可扩展）。

## 4. 待办视图定义

所有待办视图均基于**所有已加载清单**中的任务聚合。

| 名称 | key | 过滤规则 | 排序 |
|---|---|---|---|
| 今天 | `today` | 今天到期的待完成 | 按到期日升序，无到期日置底 |
| 本周 | `week` | 本周到期的待完成 | 按到期日升序 |
| 全部 | `all` | 全部待完成 | 按清单 → 分组 → 原顺序 |
| 高优先级 | `high` | 高优先级的待完成 | 按到期日升序，无到期日置底 |

## 5. 交互设计

### 5.1 侧边栏

```
待办视图
  📅 今天          12
  📆 本周           5
  🗂️ 全部          34
  🚩 高优先级       3

我的清单
  ...
```

- 当前激活的待办视图高亮显示。
- badge 显示该待办视图下符合条件的任务数量。
- 点击某个待办视图后，清除 `activeListName` 和 `activeGroup`，进入待办视图。
- 点击「我的清单」中某个清单后，退出待办视图。

### 5.2 中间区域（TodoView）

```
┌─────────────────────────────────────┐
│ 今天                          12    │  ← 标题 + 数量
├─────────────────────────────────────┤
│ 工作清单                            │  ← 清单卡片标题
│   项目Alpha                         │  ← 分组标题
│     ☐ 竞品调研报告        07-10     │  ← 任务（可勾选、可点详情）
│     ☐ 需求评审            07-11     │
│   日常事务                          │
│     ☐ 周报                07-11     │
│ 生活清单                            │
│   购物                              │
│     ☐ 买牛奶              07-11     │
└─────────────────────────────────────┘
```

- 按 `sourceList`（清单）分组，再按 `group`（分组）分组。
- 任务卡片显示来源清单标签，避免与清单视图混淆。
- 点击任务卡片任意区域打开 `TaskEditorDialog`。
- 勾选框完成任务。
- 子任务可展开/折叠并切换状态。

### 5.3 任务详情弹窗

- 标题下方显示「来自：工作清单 / 项目Alpha」。
- 分组下拉框列出该任务所属清单的所有分组。
- 保存后修改回写到原清单。
- 删除任务时提示：「确定从【工作清单】中删除该任务？」

### 5.4 空状态

如果当前待办视图没有任务：

```
🎉 当前没有符合条件的待完成
```

## 6. 数据结构改动

### 6.1 新增类型

```ts
// src/types/index.ts
export type TodoViewKey = 'today' | 'week' | 'all' | 'high';
```

### 6.2 扩展 Task

```ts
export interface Task {
  // ... 原有字段
  sourceList?: string; // 仅在待办视图聚合时使用，标识任务来自哪个清单
}
```

`sourceList` 不写入持久化文件，仅用于运行时聚合视图。

## 7. Store 改动

### 7.1 `src/stores/listsStore.ts`

新增 action：

```ts
fetchAllListsContent: () => Promise<void>
```

- 进入待办视图时调用。
- 并发拉取 `lists` 中所有清单的内容到 `fileCache`。
- 已有缓存的清单可跳过，或使用缓存后后台刷新。
- 任一清单失败不影响整体，记录警告即可。

### 7.2 `src/stores/tasksStore.ts`

#### 新增状态

```ts
todoView: TodoViewKey | null;
```

#### 新增 actions

```ts
setTodoView: (key: TodoViewKey | null) => void;
```

- 设置当前待办视图。
- 设置时自动清空 `activeListName` 和 `activeGroup`（通过 `listsStore`）。

#### 聚合任务

新增辅助函数：

```ts
function flattenAllTasks(fileCache: Record<string, ParsedList>): Task[]
```

- 遍历所有清单，给每个任务附加 `sourceList`。

修改 `getFilteredTasks`：

```ts
getFilteredTasks: () => {
  const { todoView, tasks, filter, searchQuery, sortMode } = get();
  if (todoView) {
    const aggregated = flattenAllTasks(useListsStore.getState().fileCache);
    const filtered = aggregated.filter((t) => matchesTodoView(t, todoView));
    return sortTasks(filtered, sortMode);
  }
  const filtered = tasks.filter((t) => matchesFilter(t, filter, searchQuery));
  return sortTasks(filtered, sortMode);
};
```

#### 跨清单操作路由

所有修改类 action（`updateTask`、`deleteTask`、`toggleSubtask`、`completeTaskWithoutSubtasks`、`deleteTasks`）需要支持根据 `task.sourceList` 定位清单：

```ts
function requireTaskContext(task: Task): { listName: string; list: ParsedList; saveListContent: ... } | null {
  const listName = task.sourceList ?? useListsStore.getState().activeListName;
  if (!listName) return null;
  const list = useListsStore.getState().fileCache[listName];
  if (!list) return null;
  return { listName, list, saveListContent: useListsStore.getState().saveListContent };
}
```

- 如果任务有 `sourceList`，优先使用该清单。
- 否则回退到当前激活清单（兼容清单视图）。

### 7.3 自动刷新

订阅 `listsStore.fileCache` 变化：

```ts
useListsStore.subscribe((state, prevState) => {
  if (useTasksStore.getState().todoView && state.fileCache !== prevState.fileCache) {
    useTasksStore.getState().refreshTodoView();
  }
});
```

## 8. 组件改动

### 8.1 新增 `src/components/todo-view/TodoView.tsx`

职责：
- 接收聚合后的任务数组。
- 按 `sourceList` 和 `group` 两层分组渲染。
- 渲染清单卡片和分组区域。
- 绑定完成、详情、删除、子任务切换事件。

内部结构：

```tsx
function ListSection({ listName, groups }) { ... }
function GroupSection({ name, tasks }) { ... }
export function TodoView({ tasks, ...callbacks }) { ... }
```

样式：
- 清单卡片使用轻微背景色区分（`bg-[var(--color-surface-raised)]`）。
- 分组标题使用当前 `TaskList` 的 sticky header 风格。
- 任务卡片复用现有 `TaskCard`，但隐藏拖拽手柄。

### 8.2 `src/components/layout/Sidebar.tsx`

- 待办视图配置改为：

```ts
const todoViews = [
  { key: 'today', icon: Calendar, label: '今天' },
  { key: 'week', icon: CalendarDays, label: '本周' },
  { key: 'all', icon: Layers, label: '全部' },
  { key: 'high', icon: Flag, label: '高优先级' },
] as const;
```

- 点击时调用 `tasksStore.setTodoView(key)` 并关闭移动端菜单。
- 当前 `todoView === key` 时高亮。
- 每个入口右侧显示数量 badge，通过 `tasksStore.getTodoViewCounts()` 获取。

### 8.3 `src/components/layout/ContentArea.tsx`

- 根据 `tasksStore.todoView` 决定渲染：
  - `todoView` 存在：渲染 `TodoView`。
  - 否则：渲染原有清单视图（含 Toolbar、NewTaskBar）。
- 进度条、标题等原有逻辑仅在非待办视图时生效。

### 8.4 `src/components/tasks/TaskCard.tsx`

- 可选：当 `task.sourceList` 存在时，在卡片顶部显示一个小标签：

```tsx
{task.sourceList && (
  <span className="...">{task.sourceList}</span>
)}
```

## 9. 状态流转

```
用户点击 Sidebar「今天」
    ↓
setTodoView('today')
    ↓
listsStore.fetchAllListsContent()  // 确保所有清单已加载
    ↓
tasksStore 聚合所有清单任务 → 过滤出今天到期的待完成
    ↓
ContentArea 渲染 TodoView
    ↓
用户点击任务卡片
    ↓
打开 TaskEditorDialog，sourceList = 任务来源清单
    ↓
用户保存/删除
    ↓
修改回写 sourceList 对应清单，fileCache 更新
    ↓
订阅触发，TodoView 自动刷新
```

## 10. 边界情况

| 场景 | 处理 |
|---|---|
| 用户有清单但从未访问过 | 进入待办视图时并发加载所有清单 |
| 某个清单加载失败 | 跳过该清单，显示警告，其他清单正常展示 |
| 待办视图下用户删除来源清单 | 刷新后该清单任务消失 |
| 任务被编辑后不再符合待办视图条件 | 保存后自动从当前视图消失 |
| 用户从待办视图点击清单侧边栏 | `setTodoView(null)`，回到清单视图 |

## 11. 待确认问题

1. **「全部」是否包含已完成任务？**  
   不包含。第四章已明确「全部」指「全部待完成」，即状态不等于 `done` 的任务。如果用户需要查看所有任务的历史总览，可后续新增「所有任务」视图。

2. **是否需要在待办视图显示搜索？**  
   当前方案不做。如果任务数量多，后续可添加仅按标题搜索的输入框。

3. **badge 数量是否实时反映过滤结果？**  
   是，badge 应基于当前所有已加载清单实时计算。

4. **是否同步到 URL？**  
   当前方案不同步。后续如需要浏览器回退/刷新保持状态，可追加路由 `/today`、`/week` 等。
