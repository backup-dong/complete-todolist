# 移动端适配文档

## 背景与目标

Dong Todo 桌面端体验完整，但在小屏设备上固定左侧边栏（`w-60`）会严重挤压主内容区，导致手机端几乎不可用。本文档记录移动端适配的实现方案与关键细节，方便后续维护与扩展。

目标：

- 手机端（`<md` / 默认竖屏）提供可用的任务浏览、编辑、新建、批量操作体验。
- 桌面端（`≥md` / 768px 以上）保持原有固定侧边栏 + 居中弹窗的行为不变。
- 完善 PWA 配置，使应用可安装到手机主屏。

## 改造范围

本次改造仅涉及 **UI 展示层**，不涉及数据解析、GitHub API、状态管理等核心逻辑：

- 涉及：`src/components/layout/*`、`src/components/tasks/*`、`index.html`、`vite.config.ts`、PWA 图标。
- 不涉及：`src/parser/*`、`src/github/*`、`src/stores/*`、数据类型定义。

## 响应式策略

采用 **移动优先（Mobile First）**：

- 基础类名写手机端样式。
- 使用 Tailwind `md:` 断点恢复桌面端样式。
- 主要断点为 `md`（768px），少量地方配合 `sm`（640px）做文字简化。

示例：

```tsx
<div className="flex flex-col gap-3 md:flex-row md:items-center md:justify-between">
  {/* 手机：垂直堆叠；桌面：水平排列 */}
</div>
```

## 各模块实现说明

### 1. 布局：`MainLayout` + `Sidebar` 抽屉

- 桌面端：`Sidebar` 作为固定 `w-60` 侧边栏常驻显示。
- 手机端：`Sidebar` 放入左侧抽屉，点击汉堡菜单或遮罩关闭。
- `MainLayout` 维护 `menuOpen` 状态，通过 props 传递给 `Sidebar` 和 `ContentArea`。
- `Sidebar` 接收可选 `onClose` prop，存在时显示关闭按钮，并在点击智能列表 / 清单 / 分组后自动关闭抽屉。

### 2. 内容区顶栏：`ContentArea.MobileHeader`

- 仅在小屏显示，包含：汉堡菜单（打开抽屉）、当前清单标题、进度条。
- 桌面端 `ListHeader` 内的大标题在手机上隐藏，避免重复。

### 3. 工具栏：`ListHeader`

- 手机端：操作按钮（导出周报、新建分组、批量、视图切换）横向滚动，文字在小屏简化。
- 桌面端：保持原有单行布局。

### 4. 新建任务：`NewTaskBar`

- 手机端：隐藏分组标签 pill，占位符显示分组名，新建按钮只显示加号。
- 桌面端：显示完整分组标签和「新建」文字。

### 5. 批量操作：`BatchActionBar`

- 手机端：固定底部浮层，任务列表在批量模式下自动增加底部 padding，避免最后一个任务被遮挡。
- 桌面端：保持内联底部栏。

### 6. 任务编辑弹窗：`TaskEditorDialog`

- 手机端：全屏显示（`h-full w-full rounded-none`）。
- 桌面端：居中 `h-[90vh] max-w-4xl` 弹窗。

### 7. 任务编辑表单：`TaskEditor`

- 所有双列表单项改为响应式：

```tsx
<div className="grid grid-cols-1 gap-4 md:grid-cols-2">
```

- 子任务拖拽手柄保持可见并放大到 `h-8 w-8`，未改动拖拽传感器逻辑。

### 8. 任务卡片与列表

- `TaskCard`：手机端 `p-3 md:p-4`，删除按钮固定 `h-8 w-8` 触摸区域，已兼容 `hover:none` 设备。
- `TaskList`：拖拽手柄放大，保持可见。

## PWA 配置

### `index.html`

```html
<link rel="manifest" href="/manifest.webmanifest" />
<meta name="theme-color" content="#2563EB" />
<meta name="description" content="基于 Markdown + GitHub 的待办系统" />
<link rel="apple-touch-icon" href="/icon-192.png" />
<meta name="apple-mobile-web-app-capable" content="yes" />
<meta name="apple-mobile-web-app-status-bar-style" content="default" />
<meta name="apple-mobile-web-app-title" content="Dong Todo" />
```

### `vite.config.ts`

```ts
manifest: {
  name: 'Dong Todo',
  short_name: 'Todo',
  description: '基于 Markdown + GitHub 的待办系统',
  theme_color: '#2563EB',
  background_color: '#FDFCFA',
  orientation: 'portrait',
  lang: 'zh-CN',
  icons: [
    { src: 'icon-192.png', sizes: '192x192', type: 'image/png' },
    { src: 'icon-512.png', sizes: '512x512', type: 'image/png' },
  ],
}
```

### 图标

- `public/icon-192.png`
- `public/icon-512.png`

当前为蓝色背景 + 白色字母 "D" 的临时图标，后续可替换为更精致的 Logo。

## 验证方法

### 1. 代码检查

```bash
npm run lint
npm run test
npm run build
```

### 2. Chrome DevTools 设备模式

```bash
npm run dev
```

- 切换到 iPhone SE / Pixel 7 等视口。
- 检查：侧边栏隐藏、汉堡菜单打开抽屉、任务编辑全屏、新建任务可用、批量操作底部浮层不遮挡内容。

### 3. 桌面回归

- 视口拉到 768px 以上。
- 检查：侧边栏常驻、标题栏单行、弹窗居中、无移动端元素残留。

### 4. PWA 检查

```bash
npm run build
npm run preview
```

- DevTools → Application → Manifest 检查字段完整。
- Lighthouse → PWA 审计确认可安装。

### 9. 安全区与状态栏适配

- `index.html` viewport 增加 `viewport-fit=cover`，页面可延伸至刘海屏安全区。
- iOS PWA 状态栏改为 `black-translucent`，并由 `ThemeProvider` 根据当前主题动态同步 `theme-color`（浅色 `#FDFCFA`，深色 `#121212`），避免侧边栏开关后状态栏残留灰色。
- 移动端头部、底部操作栏、侧边栏抽屉、全屏任务弹窗分别使用 `env(safe-area-inset-top)` / `env(safe-area-inset-bottom)` 内边距，防止内容被刘海、底部手势条遮挡。

## 后续可优化项

1. **拖拽体验**：当前未改传感器，仅放大手柄。如手机端拖拽与滚动冲突明显，可引入 `TouchSensor` 并配置长按触发。
2. **图标替换**：将临时 "D" 图标替换为正式应用 Logo，并补充 `purpose: maskable` 的图标变体。
3. **组件测试**：目前无组件渲染测试，后续可增加移动端抽屉/弹窗相关的 Playwright 视口测试。
