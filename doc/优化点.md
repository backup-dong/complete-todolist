# 优化点

| 优化点             | 已完成 | 备注 |
| ------------------ | ------ | ---- |
| 添加批量删除       | ✅     | 任务列表新增「批量」入口，可多选任务后一键确认删除； store 新增 `deleteTasks` action。 |
| 待办状态可多选筛选 | ✅     | 状态筛选改为 Radix DropdownMenu 多选下拉，空数组表示全部；`FilterState.status` 已改为 `TaskStatus[]`。 |
| 子任务也支持拖拽   | ✅     | 任务详情编辑区支持同层级子任务拖拽排序，新增 `reorderSubtasksAtPath` 工具函数。 |
| 将待办编辑改为弹窗，现在待办详情的编辑区域太小，字数多看起来麻烦   |   ✅   | 使用 `@radix-ui/react-dialog` 实现居中弹窗，桌面端最大宽度 `max-w-4xl`，内部可滚动；`ContentArea` 移除右侧 360px 固定面板。 |
| 子任务详情编辑做成折叠面板，每次只能编辑一个，点击另一个上一个会收缩 | ✅ | `TaskEditor` 中 `SubtaskEditor` 的展开状态提升到 `TaskSubtasksEditor`，全局最多展开一个子任务详情区；删除/拖拽时自动收起。 |
| 新增的待办，默认放在最上面 | ✅ | `tasksStore.createTask` 将新任务 `order` 设为当前分组最小 `order - 1`，并插入到分组任务数组头部。 |
| 导出周报，按完成时间顺序排列 | ✅ | `report.ts` 新增有效完成时间计算，每个分组内按任务自身或子任务（含嵌套）的最新本周完成时间升序排列；新增 `report.test.ts`。 |
| 导出添加下周计划，包含逾期任务，格式仿照完成事项，逾期可备注提醒 | ✅ | `report.ts` 新增下周计划导出：包含未完成的逾期任务与下周到期任务；按逾期优先、到期日升序排列；逾期任务标注逾期天数并附加任务备注作为提醒；无完成事项但有计划时标题自动切换为”工作周报”。 |
| 备注默认预览，点击编辑才切换 | ✅ | `NoteEditor` 默认模式从 `edit` 改为 `preview`，任务/子任务备注均默认渲染 Markdown 预览，点击顶部「编辑」按钮切换为 textarea。 |
| 去掉中间区域顶部栏的分组新增按钮 | ✅ | `ContentArea/ListHeader` 移除「新建分组」按钮，分组创建保留在 `Sidebar`。 |
| 去除底部栏新增待办左侧的分组名称显示按钮 | ✅ | `NewTaskBar` 移除左侧分组 label div，placeholder 改为「新建任务...」。 |
| 新增待办后能移动到待办位置，并高亮一下该待办 | ✅ | `createTask` 返回新任务 id；`TaskCard` 增加 `data-task-id` 与 `animate-task-highlight` 脉冲动画；`ContentArea` 创建后滚动到节点。 |
| 更新readme文档 | ✅ | 项目描述与数据结构改为 JSON 存储（兼容旧 Markdown），功能列表同步更新。 |
| 基于最新版本的json改造方案和已有文档，重新写一份2.0的todo-system-detailed-design | ✅ | 新建 `doc/todo-system-detailed-design-v2.0.md`，覆盖 JSON schema、懒迁移、持久化、UI、风险与回滚。 |
| 导出周报，子任务也需要按数字排序 | ✅ | `report.ts` 中子任务 bullet 改为同级数字编号（`1. 子任务`），保持树形前序遍历；`report.test.ts` 更新断言并新增顺序测试。 |
| 添加筛选条件清空按钮 | ✅ | `tasksStore` 新增 `clearFilters` action；`ContentArea` 工具栏在筛选/搜索非默认时显示「清除筛选」按钮。 |
| 移动端，待办列表的待办如果没有拖拽则保持水平居中 | ✅ | `TaskList` 平铺/分组容器的左侧把手 padding 仅在 `sortMode === 'drag' && !selectable` 时保留；`TodoView` 去掉无意义 `pl-7`，卡片左右对称居中。 |
| 第一次进系统，如果在加载的话，给个遮罩提示下 | ✅ | `listsStore` 新增 `initialLoading` / `listsFetched` 状态；`fetchLists` / `fetchListContent` 在首屏加载完成或失败时关闭遮罩；`AppRoutes` 额外监听 `listsFetched` + `activeListName` + `fileCache` 兜底关闭；`App` 在 `initialLoading` 为 true 时渲染全屏 `LoadingOverlay`。 |
| 移动端左侧侧滑打开侧边栏 | ✅ | `MainLayout` 添加 touchstart/touchmove 事件监听，检测左侧边缘 24px 内的向右滑动手势（≥40px 且水平距离 > 1.5×垂直距离）；保留汉堡按钮作为兜底入口。 |
| 支持分组名称编辑 | ✅ | `GroupRow` 新增「重命名」笔形按钮，点击切换为内联输入框，支持 Enter 提交/Escape 取消/失焦提交；复用 `listsStore.renameGroup`。 |
| 支持清单名称编辑 | ✅ | `ListRow` 新增「重命名」笔形按钮，点击切换为内联输入框，支持 Enter 提交/Escape 取消/失焦提交；复用 `listsStore.renameList`。 |
| 清单支持展开收缩 | ✅ | `ListRow` 左侧新增可点击的 `ChevronRight` 箭头，点击切换清单展开/折叠状态；创建新清单/新建分组时自动展开；默认折叠状态。 |
| 视图区分本周开始和本周截止 | ✅ | 待办视图新增「本周开始」选项，检查 `task.meta.start` 是否在本周范围内；原「本周」改为「本周截止」明确语义；新增 `isStartThisWeek` 日期辅助函数；`TodoViewKey` 新增 `start-week` 类型。 |
| 修复筛选下拉多选关闭问题 | ✅ | 状态筛选自定义下拉组件替代 Radix DropdownMenu，点击外部关闭，多选选项勾选时保持打开；仅选中项数 ≥2 时标签展示 `+` 连接。 |
