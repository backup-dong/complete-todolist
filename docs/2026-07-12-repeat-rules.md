# 重复任务规则调整说明

> 日期：2026-07-12  
> 影响范围：重复任务计算、任务编辑器、设置页、任务列表展示

## 背景

原有的重复规则存在语义不清晰的问题：

- `weekly` 实际上是把截止日期推到**下周一**，而不是用户直觉中的「每周同一天」。
- `weekdays` 只跳过周末，没有考虑法定节假日。
- UI 只暴露了 4 个固定选项，但后端已经支持 `mon,wed,fri`、`1,15` 这类自定义规则，用户无法直接使用。

本次调整让规则更贴近直觉，并把自定义能力暴露到界面上。

---

## 规则语义变化

| 规则值 | 旧语义 | 新语义 | 示例 |
|---|---|---|---|
| `daily` | 明天 | 不变 | 7/12 → 7/13 |
| `weekly` | 下周一 | 按**当前截止日的星期几**推进一周 | 7/3（周五）→ 7/10（周五） |
| `monthly` | 每月同一天 | 不变 | 7/15 → 8/15 |
| `weekdays` | 下一个周一到周五 | 下一个周一到周五，**并跳过法定节假日** | 9/30 → 10/9（跳过国庆长假） |
| `mon,wed,fri` 等 | 最近一个选中的星期几 | 不变 | 周三 → 周五 |
| `1,15` 等 | 最近一个选中的月内日期 | 不变 | 7/10 → 7/15 |

### 关键变化

1. **`weekly` 改为「每周同一天」**
   - 旧逻辑：完成日期是周几都无所谓，下一次一定是下周一。
   - 新逻辑：如果任务截止在周五，下一次就是下周五。
   - 这是从「固定周一」变成「固定星期几」，更符合大多数人对「每周任务」的理解。

2. **`weekdays` 跳过法定节假日**
   - 会从下一个工作日开始，逐天检查是否为周末或节假日，直到找到第一个工作日。
   - 如果节假日数据未加载或加载失败，会降级为只跳过周末。

3. **UI 支持自定义星期/日期**
   - 选「每周」后，可以勾选周一到周日的任意组合，存储为 `mon,wed,fri`。
   - 选「每月」后，可以输入 `1,15` 指定每月哪几天；留空则保持每月同一天。

---

## 数据格式

`TaskMeta.repeat` 的取值保持不变，仍然是：

```ts
'daily' | 'weekly' | 'monthly' | 'weekdays' | string
```

具体取值含义：

- `'daily'`：每天
- `'weekly'`：旧数据兼容，按当前截止日的星期几推进一周
- `'monthly'`：每月同一天
- `'weekdays'`：工作日（周一到周五，跳过法定节假日）
- `'mon,wed,fri'`：自定义每周一、三、五
- `'1,15'`：自定义每月 1 号和 15 号

**向后兼容**：

- 旧数据中 `repeat: 'weekly'` 仍然可以解析，只是语义从「下周一」变成「每周同一天」。
- 一旦在编辑器里重新保存，旧的 `'weekly'` 会被替换成具体的星期几（如 `'fri'`）。

## 节假日数据来源

### API

使用 [Nager.Date](https://date.nager.at/) 的公共节假日接口：

```
https://date.nager.at/api/v3/publicholidays/{year}/{countryCode}
```

- 默认国家代码为 `CN`（中国）。
- 会拉取当前年份和下一年份，合并去重后缓存到 `localStorage`。
- 设置页可以修改国家代码并手动刷新。

### 缓存与降级

- 节假日列表缓存在 `dong-todo:holidays`。
- 国家代码缓存在 `dong-todo:holiday-config`。
- 如果 API 请求失败、网络不通或国家代码无效，状态会变为 `error`，`weekdays` 会降级为只跳过周末。

### 调用时机

- 用户在设置页点击「刷新」时拉取。
- 完成重复任务时，如果节假日数据还没加载，会先尝试加载；已加载则直接使用缓存，不重复请求。

## UI 变化

### 任务编辑器

- 「重复规则」下拉框保留：`无 / 每天 / 每周 / 每月 / 工作日`。
- 选择「每周」后，下方显示周一到周日的开关按钮，可组合选择。
- 选择「每月」后，下方显示一个输入框，可输入 `1,15`；留空即表示每月同一天。

### 设置页

- 新增「节假日」区块：
  - 国家代码输入框（默认 `CN`）。
  - 「刷新」按钮，用于手动拉取节假日。
  - 状态提示：已加载 / 加载中 / 加载失败 / 等待配置。

### 任务卡片

- 重复规则显示从原始值（如 `mon,wed,fri`）改为中文可读文案：
  - `daily` → 每天
  - `monthly` → 每月
  - `weekdays` → 工作日
  - `mon,wed,fri` → 每周一、三
  - `1,15` → 每月1、15日

## 待确认/潜在后续

1. **节假日地区**：当前只支持单个国家代码。如果用户有跨地区需求，后续可以考虑支持多地区或自定义节假日列表。
2. **节假日 API 可用性**：Nager.Date 是公共免费 API，若在中国大陆网络环境下访问受限，可考虑切换到国内节假日接口或允许用户手动导入节假日文件。
3. **调休/补班**：当前只跳过节假日，没有处理调休补班（即把周末变成工作日）。如果需要精确模拟工作日，后续需要接入支持调休的数据源。
